Retroalimentación general

Estimados Ángela y Francisco:

El trabajo que han entregado destaca por su enfoque ordenado y el correcto uso de excepciones personalizadas. Han definido clases como LibroNoEncontrado, LibroPrestado, LibroNoPrestado y UsuarioNoEncontrado, lo que indica que han comprendido claramente los posibles escenarios de error dentro del dominio de una biblioteca.

El sistema se apoya en las clases Biblioteca y Biblioteca_DUOC para concentrar la lógica de negocio. Las clases Libro y Usuario están bien estructuradas y representan adecuadamente a las entidades del sistema. El flujo de ejecución parece estar correctamente distribuido entre estas clases, aunque no se observa una clase de interacción (menú o consola), por lo que se puede inferir que esta fue implementada dentro de Biblioteca_DUOC.

Se recomienda refactorizar algunos métodos que combinan validaciones, acciones y mensajes, para separarlos en funciones más pequeñas. Esto favorecería la reutilización de código y facilitaría la migración futura hacia otro tipo de interfaz. El uso de nombres coherentes para las excepciones y las clases denota un trabajo consciente en el diseño del sistema.

Feed Up – ¿Dónde debe llegar el estudiante?

El objetivo de esta actividad es construir un sistema funcional que maneje situaciones excepcionales mediante el uso de clases de excepción personalizadas, protegiendo el flujo mediante bloques try/catch. El programa debe aplicar principios de modularidad y orientación a objetos, así como estructuras dinámicas para almacenar y gestionar datos.

Feed Back – ¿Cómo se está desempeñando?

El desempeño ha sido bueno. Se nota un entendimiento claro del propósito de las excepciones personalizadas y su integración en el flujo del programa. Las clases están bien distribuidas y el diseño refleja una arquitectura sencilla pero funcional. Aún se puede mejorar en modularidad y separación de responsabilidades, sobre todo para preparar el sistema ante una eventual evolución futura.

Feed Forward – ¿Cómo debe seguir?

Como paso siguiente, les recomiendo implementar una clase que centralice la interacción con el usuario (como un menú o controlador), de forma que Biblioteca_DUOC solo mantenga la lógica. También pueden aplicar lo aprendido en dominios similares como inventarios, reservas o sistemas de atención, reutilizando la misma lógica de excepciones y validaciones.
