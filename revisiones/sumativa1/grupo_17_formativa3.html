Feed Up – ¿Dónde debe llegar el estudiante?

El objetivo de esta actividad es diseñar un sistema orientado a objetos que gestione una biblioteca simulada, integrando manejo de errores mediante excepciones personalizadas. Se busca que el estudiante sea capaz de anticipar fallos del sistema (como libros inexistentes o préstamos dobles) y manejar estas situaciones sin que la aplicación se detenga abruptamente, ofreciendo una experiencia informativa y controlada al usuario.

Feed Forward – ¿Cómo debe seguir?

Una buena práctica que podrías aplicar es separar la lógica de negocio de la lógica de interacción. Puedes crear una clase ServiciosBiblioteca que se encargue exclusivamente de prestar, devolver o buscar libros, dejando a la clase Main únicamente la tarea de mostrar menús o leer entradas. Esta división te permitirá reutilizar fácilmente tu lógica en otros contextos como una interfaz gráfica o una aplicación web, sin duplicar código.
