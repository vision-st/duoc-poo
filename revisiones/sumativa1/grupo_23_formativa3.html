Retroalimentación general

Estimada Dominique:

Tu proyecto refleja un entendimiento correcto de los principios de la programación orientada a objetos y del manejo de excepciones. Has estructurado las clases fundamentales como Libro, Usuario y Biblioteca de forma clara. La clase BibliotecaExceptionApp cumple el rol de punto de entrada y orquestación del sistema.

Los bloques try/catch están bien distribuidos y cumplen su función de proteger el flujo ante operaciones riesgosas.

Como sugerencia de mejora, podrías separar con mayor claridad la lógica de interacción con el usuario respecto a la lógica de negocio, para permitir una mayor escalabilidad del sistema. También sería útil encapsular algunos procesos comunes en métodos reutilizables para disminuir la duplicación de código y facilitar el mantenimiento.

Feed Up – ¿Dónde debe llegar el estudiante?

Esta actividad busca que los estudiantes desarrollen un sistema orientado a objetos capaz de gestionar errores mediante el uso de excepciones personalizadas. El programa debe usar colecciones, ser modular y proteger su flujo con bloques try/catch que entreguen mensajes informativos y eviten que el sistema se caiga ante errores comunes como préstamos inválidos o búsquedas fallidas.

Feed Back – ¿Cómo se está desempeñando?

Tu trabajo cumple con lo requerido. Las clases están bien estructuradas, las excepciones están correctamente definidas y aplicadas, y el sistema es funcional y claro. Utilizas colecciones correctamente para modelar el dominio y tu flujo es coherente. Aun así, puedes seguir mejorando la separación entre presentación y lógica de negocio para facilitar el crecimiento del proyecto.

Feed Forward – ¿Cómo debe seguir?

Como siguiente paso, podrías crear una clase separada para gestionar la entrada y salida del usuario, dejando la lógica de préstamo y validación en Biblioteca. También podrías aplicar lo aprendido en otro sistema como control de asistencia o agenda de contactos, usando la misma lógica de manejo de errores, colecciones y modularidad.
