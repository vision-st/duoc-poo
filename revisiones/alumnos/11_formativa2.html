Actividad: Utilizando Herencia y Polimorfismo

Grupo 11 - SEBASTIAN FITA ROMAN y JOSE EDUARDO ARREDONDO RIVERA

El trabajo presentado muestra una estructura clara y organizada, siguiendo correctamente los principios de la programación orientada a objetos. Se aprecia una separación efectiva en paquetes y el uso de clases auxiliares como Constants y Utils, lo que denota preocupación por mantener un código limpio y reutilizable.

La clase abstracta Cuenta cumple con lo requerido, y sus subclases aplican correctamente la sobreescritura de métodos como depositar() y girar(). El uso de una interfaz para Cliente es un buen ejemplo de polimorfismo, y los métodos implementados respetan la lógica esperada.

Se observa una estructura robusta en el flujo principal con control de excepciones y validación de entradas. Sin embargo, podrían mejorar implementando una sobrecarga real de constructores en la clase abstracta, y quizá incluir manejo de listas de múltiples clientes, en lugar de una sola instancia. También es recomendable abstraer más lógica repetitiva del Main.

4. Sobreescritura del método abstracto X Los métodos depositar y girar son sobrescritos en cada subclase con detalles propios.
5. Sobrecarga de métodos (constructores) X La clase abstracta Cuenta no posee múltiples constructores. Para lograr la sobrecarga podrían implementar:
public Cuenta(double saldoInicial) { this.numeroCuenta = generarNumeroCuenta(); this.saldo = saldoInicial; }

Feed Up

Esta actividad busca que los estudiantes dominen conceptos como herencia, polimorfismo, abstracción y uso de interfaces aplicándolos en un sistema coherente como una aplicación bancaria. Se espera también el uso consciente de estructuras organizadas y código reutilizable.

Feed Back

El código cumple con los requerimientos esenciales, demostrando comprensión de herencia y polimorfismo. Destaca el orden en los paquetes y el uso de constantes y métodos auxiliares. Falta desarrollar una sobrecarga real de constructores y considerar cómo manejar múltiples clientes a futuro.

Feed Forward

Una buena forma de reforzar lo aprendido es aplicar el mismo diseño en otro dominio, como gestión de inventario o reservas. También sería útil refactorizar el Main para mejorar la escalabilidad. Evitar la lógica repetitiva fortalecerá sus habilidades como desarrolladores.
