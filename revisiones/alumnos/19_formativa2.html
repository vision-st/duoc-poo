Actividad: Utilizando Herencia y Polimorfismo

Grupo 19 - EDGAR DARIO ESPINOZA BRAGAZZI y CAMILA INAL TAPIA

El proyecto desarrollado por el grupo cumple de forma adecuada con los objetivos propuestos para esta actividad. Se destaca el diseño centrado en la clase abstracta CuentaBancaria, que define atributos esenciales como número de cuenta y saldo, bien encapsulados y utilizados por sus subclases.

Las subclases CuentaAhorro, CuentaCorriente y CuentaCredito están implementadas de forma coherente, sobrescribiendo métodos clave como retirar o mostrarInformacion, permitiendo así comportamientos diferenciados según el tipo de cuenta. Se observa también la correcta implementación de la interfaz IMostrarDatos para estandarizar la visualización de información, lo cual fortalece el uso del polimorfismo.

El uso de sobrecarga se aprecia en la clase abstracta, permitiendo instanciaciones flexibles. La lógica del Main es clara, aunque podría beneficiarse de una mayor separación entre la lógica de negocio y la interfaz de usuario. Buen trabajo, con estilo ordenado y aplicando correctamente los conceptos.

Feed Up

La meta es implementar un sistema de cuentas bancarias utilizando herencia, interfaces, sobreescritura y sobrecarga, permitiendo estructuras flexibles y reutilizables que simulen comportamientos reales.

Feed Back

El grupo logró aplicar correctamente todos los puntos. Las clases están bien organizadas, la jerarquía de herencia es clara y el sistema es funcional. El código es legible y con buena cohesión.

Feed Forward

Como mejora futura, podrían aplicar separación de responsabilidades creando clases dedicadas a la lógica de negocio y otras a la interfaz de usuario. También se recomienda explorar el uso de colecciones más avanzadas o menús dinámicos para mejorar la experiencia de usuario.
